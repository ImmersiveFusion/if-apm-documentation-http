# Scenarios

These walkthroughs show IAPM in action - step by step through real-world troubleshooting and exploration workflows. Each scenario describes exactly what you do and what you see at every stage.

## Available Scenarios

| Scenario | Difficulty | Time | Description |
|----------|-----------|------|-------------|
| [Your First IAPM Session](first-session.md) | :material-star: Beginner | 10 min | Get oriented in the 3D environment, learn basic controls, and meet Tessa |
| [Investigating a Latency Spike](latency-spike.md) | :material-star::material-star: Intermediate | 15 min | Trace a latency spike from the Grid to root cause using Tessa and the Diagnostics Room |
| [Root Cause Analysis with Tessa](root-cause-analysis.md) | :material-star::material-star: Intermediate | 20 min | Diagnose cascading failures across multiple services with AI-assisted analysis |
| [Understanding Your Service Topology](service-graph.md) | :material-star: Beginner | 15 min | Explore your architecture in Graph view and identify traffic patterns and dependencies |

## Before You Start

Each scenario assumes you have:

- IAPM [installed](../Installation/index.md) and running
- An active subscription with a connected grid
- Familiarity with [basic navigation controls](../Navigation/index.md) (recommended for intermediate scenarios)

!!! tip "Demo Grid"
    If you don't have a production grid connected yet, you can use the demo grid to follow along with these scenarios. The demo grid includes a simulated microservice architecture with realistic traffic patterns.

## How to Read These Walkthroughs

Each step follows a consistent format:

- **What you do** - the action you take (keyboard input, mouse click, Tessa command)
- **What you see** - the visual result in the 3D environment (colors, animations, layout changes)

Look for these callouts throughout:

!!! tip "ProTip"
    Context-aware guidance that appears in IAPM itself.

!!! info "What you see"
    Descriptions of the visual feedback at each step.
